Basics

Definicje i podstawowe zasady

Rozszerzenie pliku CSS to .css.

CSS to język, w którym definiujemy style elementów HTML.

CSS - Cascading Style Sheets (Kaskadowe Arkusze Stylów).

W stylach CSS występuje hierarchia. Mogą się przesłaniać. Hierarchia w CSS nazywana jest kaskadowością.

Każdy koniec komendy w CSS musi zakończyć się średnikiem.

Można pominąć hierarchię używając !important po podaniu wartości komendy, ale przed średnikiem.

Komentarze w CSS tworzymy pisząc: /*treść komentarza*/.

Kaskadowość implikuje istnienie mechanizmu dziedziczenia cech.

Dziedziczenie - właściwości przypisane w CSS są przejmowanie z elementu nadrzędnego na bierzący. Nie dziedziczy się jednak atrybutów związanych z położeniem elementu.

Znak # oznacza id, czyli identyfikator (id z założenia jest unikalne, jedno na stronie, a przynajmniej na podstronie).

Znak . oznacza klasę. Tym różni się od id, że z założenia służy do stylowania wielu elementów na stronie.

Kod CSS piszemy w pliku .html w znacznikach <style>, w inline (dodajemy atrybut style do znacznika HTML), albo w pliku .css, który podłączamy do pliku .html linią: <link rel="stylesheet" href="style.css" type="text/css"/> (przy czym style.css to nazwa pliku .css).

Hierarchia w CSS:

1. Styl wpisany (inline, wpisany jako atrybut znacznika).

2. Styl w dokumencie (pomiędzy znacznikami <style>).

3. Styl zewnętrzny (w pliku CSS).

4. Wartość domyślna.

Przykład

Style można przypisywać do elementów na dwa różne sposoby.


									<style> 
											/*Sposób 1*/
											body
											{
													background-color: #eee;
											}
										  
											/*Sposób 2*/
											body
											{
													background-color: #eee;
											}
									</style>

Przed nazwami znaczników nie dajemy żadnego znaku (przed id powinien być hashtag, a przed klasą kropka).

Pamiętaj, że jeśli piszesz style CSS między znacznikami style w pliku .html lub w inline, to komendy się nie podświetlają!

Stylizowanie linków

Linki obsługuje znacznik <a>. W CSS możemy dodawać podklasy po drukropku. Istnieją trzy podklasy przydatne do stylizacji linków:

a:link - ustawi początkowy styl znacznika <a>.

a:visited - ustawi styl znacznika <a> (linku), który był już odwiedzony przez tego użytkownika.

a:hover - ustawi styl znacznika <a> po najechaniu.


									<style>
											a:link
											{
													color: black;
											}
											a:visited
											{
													color: red;
											}			  
											a:hover
											{
													color: green;
											}
									</style>
									
									<a href="https://www.youtube.com/channel/UCft10tZBkVSR_MkcWIK4mvg/videos" target="_blank">link</a>

Podsumowanie

Zalety stosowania plików .css:

1. Rozdzielenie stylu (CSS) od zawartości (HTML) strony.

2. Potężniejsze możliwości edycji wyglądu.

3. Uproszczenie kodu strony (brak rozbudowanych atrybutów <style> w całym dokumencie).

4. Możliwość dostosowania metod prezentacji w zależności od posiadanej przez użytkownika przeglądarki.

5. Łatwiejsza zmiana layoutu całego serwisu.

6. Spójność stylistyczna gotowej witryny (co ma wpływ na rozpoznawalność projeku w sieci).
