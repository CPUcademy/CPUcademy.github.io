Basics

CSS stands for Cascading Style Sheets. It is not a programming language. It allows us to style HTML documents. The extension of a CSS file is .css. In CSS, a semicolon is placed after each command (not everything is a command). The term "cascade" refers to how styles are applied to elements in a structured and predictable manner. When multiple styles are defined for the same element, CSS follows a set of rules to determine which style takes precedence. The hierarchy of styles can be omitted with !important (added after the command - color: blue !important;). We can create comments between /* and */ signs. Things that we write between these signs don't count as code. Comments are often used to describe the code.

We can style the HTML tags, IDs (indicated with #), and classes (marked with .). An ID has to be unique to a subpage. It is assigned to an element like this: <p id="p"></p>, and then styled in CSS: #p {...}. A class is used to style many objects on a website. An element can have many classes. They are assigned to an element like this: <p class="p p2"></p>, and then styled in CSS: .p {...}. An object can have both classes and an ID (the ID is higher in the hierarchy).

We can write CSS code inside the <style> markups in the head of the document, inline (in the style attribute of a selected element), or in an external .css file (recommended because of style integrity). We connect the external file in the head section like this: <link rel="stylesheet" href="style.css" type="text/css"/>.

CSS hierarchy:

  1. Inline styles - style="color: blue;".
  2. Styles from the <style> tag - <style>p{color: blue;}</style>.
  3. Styles from a file - p{color: blue;}.
  4. The default value.

p{color: blue;}

p - a selector

color - a property (attribute)

blue - a value

Basic commands

The all-in-one commands do not require us to provide all values but just the ones we need.

Font

font-style Setting the font style (normal, italic, oblique).
font-variant Setting the font variant (normal, small-caps).
font-weight Setting the font thickness (lighter, normal, bold, bolder, 100-900).
font-size Setting the font size (e.g., 20px).
font-family Setting the font family (Arial, Verdana, etc.).
font All-in-one in this order (separated by commas): style, variant, weight, size, family. For example: font: italic small-caps bold 16px Arial;.

Text

color Setting the text color (#303030 (hex) or white (color name)).
text-align Aligning text (center, left, right, justify).
text-decoration Decorating text (none, underline, overline, line-through, blink).
text-indent Setting the first line indentation (e.g., 5px).
text-transform Converting all text to uppercase or lowercase (uppercase, lowercase).
letter-spacing Setting the spacing between letters (e.g., 5px).
word-spacing Setting the spacing between words (e.g., 5px).
line-height Setting the spacing between lines of text (e.g., 5px).

Background

background-color Setting the background color (#303030 (hex) or white (color name)).
background-image Setting the background image (url("image.jpg")).
background-repeat Setting on which axis will the background image repeat if it is too small (repeat, repeat-x, repeat-y, no-repeat).
background-attachment Controlling whether the background image scrolls with the page (fixed, scroll).
background-position Setting the background image position by two coordinates x, y (center, top, left, right, bottom, and numerical values, e.g., 20px). We separate the two coordinates with a space. If we provide only one value, it will apply to both axes.
background All-in-one in this order (separated by commas): color, image, repeat, attachment, position. For example: background: #303030 url("image.jpg") no-repeat fixed left top;.

Sizes

width Setting the width of an element.
height Setting the height of an element.
min-width Setting the minimum width of an element. It can increase based on content, and min-height also exists.
max-width Setting the maximum width of an element. It can decrease based on content, and max-height also exists.

Border

border-width Setting the width of all / chosen borders (thin, medium, thick). We can choose specific borders for all of these commands like this: border-top-width.
border-style Setting the style of all / chosen borders (none, hidden, dotted, dashed, solid, double, groove, ridge, inset, outset).
border-color Setting the color of all / chosen borders (#303030 (hex) or white (color name)).
border All-in-one in this order (separated by commas): width, style, color. For example: border: 2px solid #303030;.
border-radius Setting how much a corner is rounded (e.g., 5px).

Lists

list-style-type Setting the look of list's numbers, dots, etc. For <ol>: decimal, decimal-leading-zero, lower-alpha, lower-greek, lower-latin, lower-roman, upper-alpha, upper-latin, upper-roman, and so on. For <ul>: circle, disc, square. The only style for both lists is none.
list-style-position Setting the position of the list's markers (outside, inside).
list-style-image Setting an image as the list's markers (url("image.jpg")).
list-style All-in-one in this order (separated by commas): type, position, image.

Margins

margin Setting the margins (outer spacings), such as distance between containers (e.g., 20px). We can choose a specific side for the margin to apply like this: margin-top.
padding Setting the paddings (inner spacings), such as text spacing from the edge of a container (e.g., 20px). We can choose a specific side for the padding to apply like this: padding-top.

Other

cursor Setting the cursor style (auto, crosshair, default, pointer, move, text, wait, help, e-resize, ne-resize, nw-resize, n-resize, se-resize, sw-resize, s-resize, w-resize).
float: left; Allows elements to appear side by side. Variants include right, inherit, and none (default).
clear: both; Clears the float effect, commonly used like this: <div style="clear: both;"></div>.
opacity: 0.8; Sets the transparency. 0 is fully invisible, and 1 is fully visible.
display: block; Makes the element a block-level element. If display: none; was used, this restores its visibility.
display: inline-block; An alternative to float.
outline: none; Removes the outline that appears when clicking a text input field (<input type="text">).
filter: brightness(70%); Adjusts the brightness of an element.

Links

We can edit how links look in different states using these selectors.

a:link A normal link
a:visited A visited link
a:hover A link hovered over at the moment
a:active A link during clicking

Zaawansowane

justify-content: space-between; - stworzy przerwę między obiektami.

z-index 100; - określa głębię. Im większa jest wartość z-index elementu, tym bliżej oczu użytkownika znajduje się obiekt.

position: fixed; - ustala stałą pozycję elementu na stronie. Przy określaniu położenia można też wykorzystać właściwości top: 0; i left: 0;. Ta komenda ma też argument relative i on ustawi "dziecku" pozycję relatywną do jego "rodzica" w kaskadzie.

box-shadow: 3px 3px 30px 5px rgba(204, 204, 204, 0.9); - ustawia cień elementu. Aby działał we wszystkich przeglądarkach, zamiast tej linii użyjemy kodu, który wygeneruje generator.

box-sizing: content-box; - domyślny sposób dla pól <input type="text"> i <input type="password"> - szerokość obliczana jest jako suma szerokości zawartości (contentu) oraz paddingu.

box-sizing: border-box; - domyślny sposób dla pola <input type="submit"> - szerokość definiowana jest w CSS i jest stała, co sprawia, że padding jest odejmowany od dostępnej, określonej w CSS przestrzeni.

flex-wrap: nowrap;

overflow-y: hidden; - usunie pionowy suwak (na całej stronie, w iframie lub innym obiekcie).

overflow-x: hidden; - usunie poziomy suwak (na całej stronie, w iframie lub innym obiekcie).

Gdy definiujemy jakieś właściwości, które nie działają we wszystkich przeglądarkach, musimy dodać przedrostki:

-webkit - Safari i Chrome.

-moz - Mozilla Firefox.

-ms - Internet Explorer.

-o - Opera.

input[type=text] - zdefiniuje wygląd dla znacznika <input> o atrybucie type równym text. Aby zdefiniować wygląd dla znacznika <input> z atrybutem type równym text lub password, piszemy:

input[type=text],

input[type=password]

*Nie występuje tu relacja dziecko - rodzic. Przecinek oznacza, że te klasy mają taki sam styl, ale nie mają ze sobą nic wspólnego w kaskadzie.

.klasa:before - tworzy pseudoelement, który staje się "pierwszym dzieckiem" danego elementu.

.klasa:first-child - zdefiniuje wygląd dla "pierwszego dziecka" klasy klasa.

input:focus - ustawi styl dla aktywnego pola <input> (nie najechanego, tylko aktywnego).

Ze względu na to, że w CSS występuje kaskadowość, możemy napisać styl dla np.: .klasa a i będzie to styl zdefiniowany dla znacznika <a> znajdującego się w klasie klasa. Można też napisać np.: ol > li > ul (zdefiniuje to wygląd dla listy <ul> będącej "dzieckiem" znacznika <li>, który jest "dzieckiem" listy <ol>). Takie kombinacje można stosować dowolnie.
