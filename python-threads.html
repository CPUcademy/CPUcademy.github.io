Threads

Threads are a fundamental part of the Python concurrency model, allowing multiple tasks to run simultaneously within a single program. By utilizing threads, we can create more responsive and efficient applications, particularly in scenarios where tasks can be performed in parallel. Python provides built-in support for multithreading through the threading library.

To create a thread, we instantiate a Thread object and select a function (its target), which contains the code to be executed by the thread. Once the thread is created, it can be started using the start() method.


import threading

def count_to_10000():
    for i in range(0, 10000):
        if i == 9999:
            print("Task1: Counted to", i)

def sum_to_10000():
    total = sum(range(0, 10000))
    print("\nTask2: Sum = " + str(total))

# Creating threads
task1 = threading.Thread(target = count_to_10000)
task2 = threading.Thread(target = sum_to_10000)

# Starting threads
task1.start()
task2.start()

# Waiting for both threads to finish
task1.join()
task2.join()

print("Both tasks completed.")

Threads can be synchronized to prevent concurrent access to shared resources, ensuring data consistency. Synchronization is applied using the Lock object. It ensures that only one thread accesses shared resources at a time. Below, you can see an improved version of the code above. Check the difference in displaying the data between these two programs in your editor.


import threading

# Creating a lock for synchronization
lock = threading.Lock()

def count_to_10000():
    for i in range(0, 10000):
        if i == 9999:
            # Acquiring the lock before printing
            with lock:
                print("Task1: Counted to", i)

def sum_to_10000():
    total = sum(range(0, 10000))
    # Acquiring the lock before printing
    with lock:
        print("Task2: Sum = " + str(total))

task1 = threading.Thread(target = count_to_10000)
task2 = threading.Thread(target = sum_to_10000)
task1.start()
task2.start()
task1.join()
task2.join()

print("Both tasks completed.")

GIL (Global Interpreter Lock)

The Global Interpreter Lock (GIL) is a mechanism used in CPython (the most widely used Python interpreter) to ensure that only one thread executes Python bytecode at a time in a single process. You can't "use" it directly because it is an internal mechanism in the CPython interpreter that runs automatically. If our program is CPU-bound and we want to take full advantage of multiple CPU cores, using the multiprocessing module (library) can bypass the GIL. This module spawns separate processes, each with its own interpreter and GIL, allowing true parallelism.


import multiprocessing

# A simple CPU-bound task
def cpu_bound_task():
    total = 0
    for i in range(10 ** 7):
        total += i
    return total

processes = []
for x in range(4):
    process = multiprocessing.Process(target = cpu_bound_task)
    processes.append(process)
    process.start()

# Waiting for all processes to finish
for process in processes:
    process.join()

print("Completed all processes.")
