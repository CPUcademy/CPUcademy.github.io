Introduction to MySQL

SQL is a programming language created for managing and manipulating data within relational databases. A relational database structures information into one or more tables, where each table consists of data arranged in rows and columns.

After creating an ERD diagram with DBDesigner, we can export it as an SQL script. This way, we have the structure ready without writing any code and can start inserting data immediately.

If the database is to be imported and exported to a file, this file has to contain these lines at the top:


CREATE DATABASE IF NOT EXISTS database_name;
USE database_name;

mysql -u root -p < [path] - importing a database from a file.

mysqldump -u root -p --routines --triggers --events --add-drop-database --create-options --databases [database_name] > [path_to_the_output_file] - exporting a database to a file.

chcp 65001 - enabling UTF-8 formatting in the console.

mysql -u root -p [database_name] / mysql -u root -p and use [database_name]; - entering the database from an RDBMS terminal.

SHOW databases; - displaying available databases in an RDBMS terminal.

SHOW tables; - displaying all tables.

DESCRIBE table_name; - displaying the structure of a given table.

LOAD DATA INFILE [path] INTO TABLE table_name FIELDS TERMINATED BY '\t' LINES TERMINATED BY '\n' IGNORE 1 LINES (id, value1, value2); - loading the data from a file into the database. The data has to be separated by TABs (\t), and the end-of-line sign must be an enter (\n). The first line will be ignored (the column titles).

EXIT; - exiting the database in an RDBMS terminal.
