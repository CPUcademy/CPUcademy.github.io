Form controls

The <form> tag is used to create interactive controls that users can use to enter data and submit to a web server for processing. It is a fundamental part of web development, allowing users to interact with a website, such as filling out contact forms, login forms, and surveys. A basic form structure looks like this:


<form action="submit_form.php" method="post">
    <!-- Form elements go here -->
</form>

The action attribute specifies the URL to which the form data is sent. The method attribute defines how data is sent. Common values include post (to send data in the body of the request) and get (to append data to the URL).

Form Input Types

The different values of the type attribute in the <input> tag:

  • text
    • Defines a single-line text input field for user input.
    • <input type="text" name="username" placeholder="Enter your username">
    • Instead of placeholder, we can use value. Check the difference by trying to enter some text.
  • password
    • Defines a password input field. Characters typed are obscured.
    • <input type="password" name="password" placeholder="Enter your password">
  • email
    • Defines a field for entering an email address. It can validate email format automatically (if contains @, etc.).
    • <input type="email" name="email" placeholder="Enter your email">
  • number
    • Defines a field for numeric input. You can set a range using the min and max attributes. An optional step attribute defines the increments between valid values (e.g., step="0.1" for decimal values).
    • <input type="number" name="age" min="18" max="100" placeholder="Enter your age">
  • checkbox
    • Defines a checkbox, allowing users to select one or more options.
    • <input type="checkbox" name="subscribe" value="yes"> Subscribe to newsletter
    • Subscribe to newsletter
  • radio
    • Defines a radio button. It allows users to select one option from a group.
    • <input type="radio" name="gender" value="male"> Male
    • <input type="radio" name="gender" value="female"> Female
    • Male
    • Female
  • file
    • Defines a file upload button that allows users to select and upload a file.
    • <input type="file" name="profile_pic">
  • submit
    • Defines a button for submitting the form data.
    • <input type="submit" value="Submit">
  • reset
    • Defines a button that resets all form fields to their default values.
    • <input type="reset" value="Reset">
  • button
    • Defines a generic button for use with JavaScript (often for custom form actions).
    • <input type="button" value="Click me" onclick="alert('Button clicked!')">
    • This version works the same. If we do not add the type="button", it will work like the submit button.
    • <button type="button" onclick="alert('Button clicked!')">Click me</button>
  • textarea
    • Defines a multi-line text input field, often used for comments or messages. To block resizing (in CSS): resize: none;.
    • <textarea name="message" rows="4" cols="10" placeholder="Enter your message"></textarea>
  • select <option>
    • Defines a drop-down menu for selecting one or more options.
    • 
<select name="country">
    <option value="USA">USA</option>
    <option value="Canada">Canada</option>
    <option value="UK">UK</option>
</select>

Radio button

Radio button to guzik do odznaczenia (możemy zaznaczyć tylko jeden z podanych). Aby go uzyskać przypiszemy do type wartość radio. Jednak zazwyczaj taki przycisk ma etykietę. Dodajemy ją za pomocą znacznika <label>. Są dwa sposoby powiązania etykiety z przyciskiem:
Sposób 1: <input type="radio" id="idd"><label for="idd">etykieta<label>.
Sposób 2: <label><input type="radio">etykieta</label>.

Tak jak napisałem, można wybrać tylko jeden z dostępnych przycisków tego typu. Ale skąd przeglądarka ma wiedzieć, z której grupy tych przycisków można zaznaczyć tylko jeden (przecież takich grup może być w jednym formularzu wiele)? Należy dodać atrybut name do przycisków i te, które są ze sobą powiązane, przypisać do tej samej nazwy.

Lista wyboru

Chodzi o rozwijaną listę wyboru. W tym przypadku nie wykorzystamy znacznika <input>. Tworzymy ją tak:


                                    <select>
                                            <option>Opcja 1</option>
                                            <option>Opcja 2</option>
                                    </select>

Znacznik <fieldset>

Wszystko co znajduje się pomiędzy tymi znacznikami, jest w ramce. Jest on często stosowany w formularzach, aby oddzielić od siebie poszczególne sekcje. Jeśli chcemy nadać tytuł tej ramce, to dodajemy w środku znacznik <legend> i pomiędzy otwierającym a zamykającym piszemy tytuł.

Pole wyszukiwania

Jest to zwykłe pole tekstowe poza tym, że ma znak "x" po prawej stronie. Służy on do czyszczenia okienka. Pole wyszukiwania tworzymy nadając argumentowi type wartość search.

Okno na datę

Ustawia się je nadając atrybutowi type wartość date (można wybrać datę z graficznego kalendarza). Nie działa to we wszystkich przeglądarkach np.: w IE.

Okno na czas

Ustawia się je nadając atrybutowi type wartość time. <input type="time" min="09:00" max="18:00"> - atrybuty min i max ustawiają zakres czasu, z którego możemy wybrać godzinę. Można też dodać atrybut step. Nie działa to we wszystkich przeglądarkach np.: w IE.

Pole na numer telefonu

Pole to nie sprawdza, czy podany numer telefonu istnieje, tylko czy jego składnia jest poprawna (zawiera tylko liczby itd.). Tworzymy go nadając atrybutowi type wartość tel.

Pole na adres URL

Pole to nie sprawdza, czy podany adres internetowy istnieje, tylko czy jego składnia jest poprawna (zawiera https:// itd.). Tworzymy go nadając atrybutowi type wartość url.

Info

W podanych powyżej trzech przykładach występuje przeglądarkowa prewalidacja formularza. Nigdy nie może stanowić właściwej sanityzacji wejścia po stronie serwera! Nie możemy ufać front-endowym ograniczeniom nałożonym w interfejsie. W przeglądarkach, które nie obsługują tych pól nastąpi tzw. fallback (cofnięcie się, downgrade) do zwykłego inputa typu text (a więc pozbawionego prewalidacji).

Aby pobrać wartości z danej kontrolki przypisujemy do niej atrybut name i wpisujemy nazwę, aby potem skorzystać z niej w PHP.

Color picker

Tworzymy go nadając atrybutowi type wartość color. Często jest zastępowany przez color picker z jscolor.com, dlatego że nie działa np.: w IE.

Dodatkowe atrybuty kontrolek

required - kontrolka staje się obwiązkowa do wpisania / zaznaczenia.

disabled - kontrolka staje się zablokowana.

readonly - elementowi formularza nie można zmienić wartości.

multiple - wtedy będzie można podawać dwie wartości (np.: po przecinku). Po dodaniu do listy rozwijanej staje się ona od razu rozwinięta, a jeśli dodatkowo dodamy do niej atrybut size, to możemy przypisać liczbę wierszy, która ma być widoczna bez rozwijania. Po zastosowaniu tego atrybutu musimy też w name tej listy rozwijanej dopisać [] np.: nazwa[]. Oznaczać to będzie, że wartości zapiszą się do listy.

checked - przycisk będzie domyślnie zaznaczony (ale będzie go można odznaczyć). Przy liście rozwijanej wpisujemy selected zamiast checked (pamiętaj, aby przy liście rozwijanej dodać to jako atrybut do tagu <option>, a nie do <select>).

Progressbar

Stworzy pasek pokazujący np.: procent naładowania baterii. Tworzymy go w ten sposób: <progress value="75" max="100"></progress>. Atrybut value określa aktualny procent zapełnienia paska, a max - ile procent może być maksymalnie. Czasami jest zastępowany przez: ProgressBar.js lub LoadingBar.js.

Lepszą alternatywą <progress> jest znacznik <meter>. Różni się tym, że możemy ustawić kolor paska. Tworzymy go tak:
<meter min="0" max="100" title="procent" value="30" low="30" high="70" optimum="75"></meter>. Atrybuty min i max określają minimalny i maksymalny poziom paska, value określa aktualną wartość, low i high określają kiedy poziom jest wysoki, a kiedy niski (poniżej min - pasek jest czerwony, powyżej min - pasek jest źółty, a powyżej high - pasek jest zielony). optimum określa kiedy poziom jest optymalny, a title - co pojawi się po najechaniu myszką na pasek.

File

Służy on do pobierania plików od użytkownika. Tworzymy go nadając atrybutowi type wartość file. Możemy dodać też atrybut accept i podać rozszerzenia plików jakie akceptujemy (użytkownk może to pominąć wybierając "Wszystkie pliki" z listy w eksploratorze plików). Ta kontrolka sprawia dużo problemów.

Problemy back-endowe

Należy dokładnie sanityzować plik przesłany przez użytkownika. Pozwolenie na umieszczenie na naszym serwerze pliku to ryzykowna pod kątem bezpieczeństwa decyzja. Odebranie wartości w PHP następuje z użyciem tablicy $_FILES (tym razem jest ona dwuwymiarowa).

Problemy front-endowe

Do ostylowania kontrolki najczęściej używamy okrężnych metod - samego inputa ukrywa się poprzez dopisanie display: none;, natomiast rolę wizualizacji tej kontrolki przejmuje odpowiednio ostylowana etykieta (znacznik <label>).

Inne opcje

Atrybutowi type w znaczniku <input> można przypisać jeszcze: month, week, datetime, datetime-local, range i hidden.
