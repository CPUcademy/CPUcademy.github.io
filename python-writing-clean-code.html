Testy jednostkowe

Testy jednostkowe służą do sprawdzania czy kod działa poprawnie. Do ich uruchomienia potrzebny jest PyCharm.

Napisałem klasę "Obliczenia", która oblicza silnię. Niżej napisałem testy sprawdzające czy działa poprawnie. Komenda assertEqual sprawdza czy metoda wywołana dla jakichś wartości zwróci wartość podaną po przecinku, a komenda assertGreater sprawdza czy metoda wywołana dla jakichś wartości zwróci wartość większą niż tę podaną po przecinku (oczywiście jest więcej takich komend). Testy odpalamy klikając na zielone strzałki na marginesie.

Magic Numbers

Magic Numbers to liczby używane w kodzie bez żadnego wyjaśnienia ich znaczenia np. parametry widżetu w bibliotece Tkinter. Ich użycie sprawia, że program jest mniej czytelny, a tym samym trudniejszy w debugowaniu i aktualizacji. Proces ich usuwania nazywany jest refaktoryzacją kodu i polega na zastąpieniu powtarzających się liczb zmiennymi zdefiniowanymi na górze dokumentu.

Dokumentowanie aplikacji

Dokumentacja służy do opisywania funkcji, klas itp. Występuje w programach (tym się zajmiemy) i na stronach internetowych (np. dokumentacja internetowa Pythona).


									#Zamknięcie tekstu w potrójnych cudzysłowiach sygnalizuje programiście, że jest to dokumentacja kodu.
									#Dokumentację danej funkcji można wyświetlić za pomocą __doc__.
									def funkcja(a, b, c):
											"""Ta funkcja przyjmuje trzy liczby, a potem zwraca ich sumę."""
											return a + b + c
									  
									print(funkcja(1, 2, 3))
									print(funkcja.__doc__)

									#Dzięki temu zapisowi można też wyświetlać tekst w różnych linijkach używając jednej komendy print().
									print("""Wonsz
									Python
									HTML
									CSS""")

Funkcja trim pozwala na pisanie dalszej części dokumentacji w tym samym bloku co jej pierwsza linijka (inaczej będzie tam "tab"). Ta funkcja jest dostępna w dokumentacji internetowej Pythona, ale jest w niej kilka błędów, więc udostępniam ją poprawioną tutaj.
