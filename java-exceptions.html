Exceptions

If we want to prevent errors, we can use the try and catch blocks. If inside the try block, an exception will be thrown (an exception will be returned through the throw command) or it will occur unforeseen (e.g., someone will divide by 0), the rest of the instructions inside this block will be skipped, and the catch block for a given exception type will execute (it will catch this exception). If inside the parenthesis, instead of the specific exception type the catch block is supposed to look for (e.g., ArithmeticException), we will type Exception, it will execute for the exception types that don't have their own specified catch block. We can add a third block - finally. It will execute regardless of what happens in try or catch.

In the example below, the division() function throws an exception of a DivisionException type (our class's type) when someone tries to divide by zero. This exception will be caught, and the getErrorMessage() method from this class will be called (it will display an error message).


public class Main
{
    static double division(double x, double y) throws DivisionException // "throws" says what object this function may (but doesn't have to) return
    {
        if(y == 0)
        {
            throw new DivisionException("Don't divide by 0!");
        }
        return x / y;
    }
    public static void main(String[] args)
    {
        int x = 10;
        try
        {
            if(x == 10)
            {
                throw new Exception("x equals 10");
            }
            System.out.println("x");
        }
        catch(Exception e) // it could be also: ArithmeticException, etc.
        {
            System.out.println("General exception: " + e.getMessage());
        }
        finally
        {
            System.out.println("I am executed regardless of anything.");
        }

        System.out.println();

        try
        {
            double result = division(5, 0);
        }
        catch(DivisionException e)
        {
            e.getErrorMessage();
        }
    }
}

class DivisionException extends Exception
{
    public DivisionException(String string)
    {
        super(string);
    }
    public void getErrorMessage()
    {
        System.out.println("Don't divide by 0!");
    }
}

The assert keyword

The assert keyword checks whether a given condition is true, and if not, throws an AssertionError. To enable assertions, the -ea (or -enableassertions) JVM option must be used when running the program.


int number = -5;
assert number > 0 : "The number must be positive.";
System.out.println("Number is: " + number);
