Error handling

Obsługa błędów

Dzięki metodzie try, except możesz zapobiec wyskoczeniu błędu (definiując co ma się stać, jeśli coś pójdzie źle).


									try:
											x = int(input("Podaj liczbę: "))
									except:
											print("Wystąpił błąd!")
											x = int(input("Spróbuj ponownie: "))

Instrukcja try, except ma jeszcze trzeci, opcjonalny blok - finally.


									try:
											file = open("test.txt", "w", encoding="utf8")
											file.write("tekst")
									except:
											print("Coś poszło nie tak!")
									finally:
											file.close()

Ten blok wykona się bez względu na to, czy wykona się try, czy except.

Słowo kluczowe raise

To słowo kluczowe służy do wywołania konkretnego typu błędu.


									x = 0
									if x == 0:
											raise ZeroDivisionError("Nie można podzielić.")

Debugowanie - breakpoint

Jeśli wyskakuje nam błąd i nie umiemy go naprawić to zamiast Run, klikamy Debug. Po odpaleniu debuggera, jeśli wiemy, w której linijce jest błąd (ale nie wiemy jaki) to ustawiamy tam breakpoint. Ustawia się go, klikając w margines przy tej linii (breakpoint to czerwona kropka). Można go używać w różnego rodzaju edytorach, ale w Python IDLE nie.

Debugger zatrzymuje kod, żeby można było wykryć błąd (pokazuje zawartość zmiennych itd.). Aby posuwać się w kodzie klikamy F8 na klawiaturze lub naciskamy przycisk "Step Over" na ekranie (w edytorze PyCharm).

Biblioteka logging

Biblioteka logging umożliwia wyświetlanie komunikatów na ekranie podczas działania programu. Patrząc na komunikaty, możemy wywnioskować kiedy coś się popsuło i to naprawić. Aby zatrzymać pojawianie się komunikatów, możemy zastosować komendę logging.disable().


									import logging
									logging.basicConfig(level=logging.DEBUG, format=" %(asctime)s - %(levelname)s - %(message)s")
									logging.debug("Początek programu")

									def silnia(n):
											logging.debug("Początek wywołania funkcji silnia(%s%%)" % (n))
											total = 1
											for x in range(n+1): #Żeby naprawić ten program, tutaj piszemy "for x in range(1, n+1):".
													total *= x
													logging.debug("x wynosi" + str(x) + ", wartość całkowita wynosi " + str(total))
											logging.debug("Koniec wywołania funkcji silnia(%s%%)" % (n))
											return total

									print(silnia(5))
									logging.debug("Koniec programu")

Raise

Słowo kluczowe raise służy do zgłaszania błędów różnego typu.


									raise Exception("Błąd")
									raise SyntaxError("Błąd")
									raise OSError("Błąd")
									#Jest jeszcze wiele innych rodzajów błędów, ale nie wypiszę ich wszystkich.

Słowo kluczowe assert służy do debugowania. Pozwala na przetestowanie, czy postawiony warunek zwraca wartość True, a jeśli nie, to zgłasza AssertionError.


									liczba = 5
									assert liczba < 0, "liczba powinna być mniejsza niż 0"

IOError

IOError sprawdza, czy wystąpił błąd przy zapisywaniu bądź wczytywaniu pliku.


									try:
											file = open("test.txt", "w", encoding="utf8")
											file.write("tekst")
											file.close()
									except IOError: #IO error
											print("Nie można utworzyć pliku na dysku!")
