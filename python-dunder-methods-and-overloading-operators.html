Dunder methods and overloading operators

__init__() (initialization) is an example of a special method. These methods are called dunder (double underscore). Some overload operators and others allow us to use Python's built-in functions.

The most important dunder methods

__call__() (callable)

It allows an object to be called as a function.


class Addition:
    def __init__(self, x):
        self.x = x

    def __call__(self, y):
        return self.x + y

add = Addition(5)
result = add(10)
print(result)

__str__() (string)

It defines what will be displayed if our object is used in the print() command.


class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        return f"{self.x} {self.y}"

point = Point(1, 2)
print(point)

__repr__() (representation)

It is a version of the __str__() method designated for displaying more technical data.


class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"Point({self.x}, {self.y})"

point = Point(1, 2)
print(repr(point))

__import__

It imports libraries based on a string, allowing for dynamic module loading at runtime, which is useful for situations like plugin systems or when the module to be imported depends on user input or configuration files.


names = ["random"]
for x in names:

__dict__

It converts class objects into dictionaries.


class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(1, 2)
print(p.__dict__)

Overloading operators

Overloading operators means overwriting the operator's functionalities so it can work with, e.g., our class's objects. In Python, we cannot overload operators directly as we do in languages like C++ (e.g., ==, <<). Here, we overwrite methods corresponding to the operators (e.g., __eq__() equals ==). We can overload operators thanks to the dunder methods listed below:

__add__ - addition

__sub__ - subtraction

__mul__ - multiplication

__truediv__ - division

__floordiv__ - floor division (integer division)

__mod__ - modulo division

__pow__ - exponentiation

__eq__ - equal to (==)

__ne__ - not equal to (!=)

__gt__ - greater than (>)

__lt__ - lesser than (<)

__ge__ - greater than or equal to (>=)

__le__ - lesser than or equal to (<=)

__eq__() (equal to)

It defines the behavior of equality comparison between our objects (now we can compare our classes by chosen values and not ID addresses).


class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

point1 = Point(1, 2)
point2 = Point(1, 2)

print(point1 == point2)

__add__() (addition)


class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Point(self.x + other.x, self.y + other.y)

point1 = Point(1, 2)
point2 = Point(3, 4)
result = point1 + point2
print(result.x, result.y)

__lt__() (lesser than)


class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __lt__(self, other):
        return self.x < other.x and self.y < other.y
