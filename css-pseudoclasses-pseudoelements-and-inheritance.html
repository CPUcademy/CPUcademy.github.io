Pseudo-classes, pseudo-elements, and inheritance

Pseudo-classes

A pseudoclass is an attribute of an attribute. It looks, e.g., like this: :hover. Thanks to them, we can, e.g., edit how links look in different states.

a:link A normal link
a:visited A visited link
a:hover A link hovered over at the moment
a:active A link during clicking 

<style>
    a{color: black;}
    a:hover{color: red;}
</style>

We style each of these states as a separate attribute. To style form controls, we use pseudoclasses like :checked or :disabled. The :focus pseudoclass indicates, e.g., an input text control that the user is using now.

:first-child and :last-child

These two pseudoclasses indicate the first and the last element of a certain group.


<style>
    p:first-child{color: red;}
    p:last-child{color: blue;}
</style>

<div>
    <p>Lorem ipsum</p>
    <p>Lorem ipsum</p>
    <p>Lorem ipsum</p>
</div>

:nth-child

This pseudoclass indicates the even or odd elements of a certain group. It can also indicate a specific element.


<style>
    p:nth-child(even){color: red;}
    p:nth-child(odd){color: blue;}
    p:nth-child(3){color: green;}
</style>

:not

This pseudoclass styles elements that do not have a certain class assigned to them.


<style>
    p:not(.r){color: green;}
    .r{color: red;}
</style>

<div>
    <p class="r">Lorem ipsum</p>
    <p>Lorem ipsum</p>
    <p class="r">Lorem ipsum</p>
</div>

Pseudo-elements

.class:before creates a pseudo-element that becomes the "first child" of the specified element.

Inheritance

Because CSS follows a cascading structure, we can define a style for, for example, .class a, which specifies styles for the <a> tag inside an element with that has a class named .class. Similarly, we can write: ol > li > ul. This defines styles for a <ul> list that is a "child" of an <li> tag, which in turn is a "child" of an <ol> list. Such combinations can be used freely to target specific elements in your CSS.
