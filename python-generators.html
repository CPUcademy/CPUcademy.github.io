Generators

An iterator is an object that contains a countable number of values. The next() function is a built-in function used to retrieve the next item from an iterator. We can use this function to advance the iterator to the next item and return its value.


iterator = iter([1, 2, 3, 4, 5]) # iter() is a built-in function that creates an iterator from an iterable object

print(next(iterator))
print(next(iterator))
print(next(iterator))

A generator is a special type of iterable object that allows us to iterate over a sequence of items without needing to store the entire sequence in memory at once. It is defined using functions that contain one or more yield statements instead of return.

When the yield keyword executes, it pauses the execution of the function, returns a value to whatever is iterating through the generator object, and comes back to the function. The two examples below have the same output, except the last value of the second program is 4 and not y, because the next() function ends the iteration once it completes the last yield. When you’re manually calling next(), the function halts after yielding all its values and doesn’t proceed to the print("y") statement. The loop going through the generator, on the other hand, handles the entire iteration and reaches the final print("y").


def generator(n):
    for x in range(n):
        print("x")
        yield x
        print("y")

for y in generator(5):
    print(y)
                                    

def generator(n):
    for x in range(n):
        print("x")
        yield x
        print("y")

x = generator(5)
for i in range(5):
    print(next(x))

When a generator function is called, it doesn't execute immediately. Instead, it returns a generator object that can be iterated over. Each time the next() function is called on the generator object, the function's execution resumes from where it left off, continuing until a yield statement is encountered. The value yielded by the yield statement is returned by the next() function, and the function's state is saved until the next call to next().

Practical example

The use case of a generator is such that we can loop through a sequence or a large amount of data without needing to store all of it. We use it when we do not care about the data before and after the particular iteration.


def read_large_file(file_path):
    with open(file_path, "r") as file:
        for line in file:
            yield line

def process_line(line):
    if "x" in line:
        print(line)

for line in read_large_file("huge_file.txt"):
    process_line(line)
