Basics

Fundamental rules

Python counts from zero, not from one.

Quotation marks and apostrophes mean the same, although we can't close one string between a quotation mark and an apostrophe.

In Python, we don't have to end a line with a semicolon like in different languages. We also don't have to indicate which commands belong to, e.g., a loop, in any other way than indentations (properly placed TABs).

We can create comments by using the # symbol. Things that we write after this sign don't count as code. Comments are often used to describe the code. Multiline comments can be written between """ and """ signs.

Everything in Python is an object.

Variables

A variable is a "drawer" for data in the RAM. We can store every sort of data in them. In Python, we have four types of variables:

int - integers

float - decimals

str (string) - texts (a number can also be a part of it, e.g., "abc123")

bool (boolean) - True (all numbers except 0) and False (0) values

In Python (unlike in, e.g., C++), we do not have to define the type when creating a variable. It is assigned automatically based on the value.

Everything on the left side of the = sign is a variable's name (it cannot contain spaces), and on the right is a value.

Be cautious because, e.g., variables x and X are not the same!

While naming the variables, we cannot add any numbers or spaces at the beginning. The first letter of the variable should not be uppercase. We should write variable names in one of these three ways: prime_number, primeNumber, nPrimeNumber.

A constant is a variable whose value cannot be changed. We can't define a variable like this in Python, but by convention, we write variables that are supposed to be constant in uppercase.

\n symbolizes the enter key and can be used inside of strings.

Some PEP8 standard guidelines (you can come back here later):

  • Functions and classes should be separated by two blank lines.
  • In a class, methods should be separated by one blank line.
  • A line should be at most 79 characters long.
  • Stay consistent with your style (spaces, etc.)
  • And an extra: Always assume the user of your code is stupid and validate your input in every way possible.

*In the examples I provide, I will not always give the entire code but fragments of it.

The print() command

The print() command displays data of all types on the screen.


# Printing text
print("Hello World!")

# Printing variables
x = 10
print(x)

# Mixing elements
y = 10
print(5, "+ 5 is equal", y)

# Printing text inside of quotation marks
print("\"Text\"")

# Printing text after an enter
print("\nText")

# Concatenation (joining strings)
print("a" + "b")

The input() command

The input() command retrieves data from the user and saves it to a variable.


x = input()
print(x)

# Adding a text that will be displayed before the answer field
x = input("Enter some data: ")
print(x)

We can add an intended type to input. It is a form of pre-validation. If the user inputs a value of a different type than the suggested one, an error will occur.


x = int(input("Enter an integer: "))
