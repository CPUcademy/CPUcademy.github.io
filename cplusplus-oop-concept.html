OOP concept

OOP stands for Object-Oriented Programming. In OOP, we use classes which are templates that define the structure and behaviour of objects. Objects are the basic units of OOP. A class is a mold for creating objects. Functions that are created inside classes are called methods. When we assign a class to a variable, we can use the methods of this class by referring to this variable. Attributes are variables associated with an object. An instance of a class is an object that belongs to that class.

Structures should be used to store data that describes something. We should use classes when we are adding the so-called functionalities or activities for such objects (methods).

To work with classes, we can create three files inside a project: the main one, another .cpp file, and a .h file. In the .h file, we write the class itself, in the another .cpp file, we write the definitions of the class's methods, and in the main one - everything else. We can write everything in one file, but if you want to write in separate files, remember to include the file.h file in the other files (#include "file.h").

If we want to make our code clean, inside the class, we write only the object's declarations. Their definitions should be written after exiting the class like this: Human::getAge() (this is the getAge() method from the Human class). The :: operator symbolizes affiliation to the class. I will not write like this to save space and mitigate redundancy in the examples.

Everything that is inside of a class can, by default, be accessed only inside of it (private). We can make it public or protected (or, again, private) by typing these words with colons. Everything that is after that label can be accessed in this particular way (until encountering an another one). public means that the objects are also available outside of the class. protected means that the objects are private, except for the class's subclasses (I will explain this in a lesson about inheritance). These modes are called access modifiers.

Abstraction

Abstraction deals with complexity by hiding unnecessary information from the user. Data is visible only to semantically related functions so as to prevent misuse. In this approach, we use the public mode often. Example: We can do many things on a computer, but it doesn't show us the inside process of how it's doing them. The implementation parts are hidden. It is the first concept of object-oriented programming.

Constructor (NameOfTheClass()) is a method that is called by the compiler when the object of the class is created. A destructor (~NameOfTheClass()) is called by the compiler when the object is destroyed. In C++, they are mandatory (but we can leave them empty). The constructor is commonly used for setting the initial values of variables. These two methods never feature the return instruction. Methods can also take arguments. If a constructor takes arguments, we give them while assigning the class to a variable (e.g., Human x(5);). A class can have a few constructors that execute accordingly to the number of given arguments, e.g., if we give three arguments when creating a new instance, the constructor that takes three arguments of that specific type is called.

A field is a variable of any type that is declared inside the class. In a class, the order of methods and other objects is irrelevant because all objects have access to the whole class the whole time.

A getter is a method that is used to return values outside of a class, and a setter is a method that edits an object of the class. We should use them because, inside them, we can add, e.g., validation rules.

The -> operator indicates that an object belongs to a given class. this keyword can replace the class name while using the -> operator. It indicates the class that it is used in.


#include <iostream>
using namespace std;
			
class Human
{
	public:
		Human()
		{
			age = 0;
			cout << "Constructor" << endl;
		}
		
		Human(unsigned short age)
		{
			this -> age = age;
		}
		
		~Human()
		{
			cout << "Destructor" << endl;
		}
		
		unsigned short getAge()
		{
			return age;
		}
		
		void setAge(unsigned short age)
		{
			this -> age = age;
		}
		
		unsigned short age;
};

int main()
{
	Human h; // an instance of a class
	h.age = 20;
	h.setAge(10);
	cout << h.getAge() << endl;
	
	Human h2(40);
	cout << h2.getAge() << endl;
		
	Human *h3 = new Human[3];
	for(int i = 0; i < 3; i++)
	{
		h3[i].setAge(30 + i);
		cout << h3[i].getAge() << endl;
	}
	delete[] h3;
	return 0;
}

Encapsulation

Encapsulation prevents external code from being concerned with the internal workings of an object. It involves hiding the fields and methods of a class so that they are not accessible from the outside but only inside of the class. The difference between abstraction and encapsulation is that abstraction hides objects and processes to show only the result at the end because it is the only thing that matters, while encapsulation hides them and blocks access to them, e.g., because some data should not be changed during the processes. It is the second concept of object-oriented programming.

If an object is created in private mode, it means that it cannot be called outside its class. Encapsulation is used to block the possibility of bypassing certain instructions by, for example, editing the output variable outside the class. Most objects should be private, and outside of the class they should be edited only through methods of this class. If we try to call or change the value of a private object outside the class, we will get an error.


#include <iostream>
using namespace std;
			
class Bank
{
	// private by default
	int balance;
	void setBalance(int balance)
	{
		this -> balance = balance;
	}
		
	public:
		Bank()
		{
			balance = 1000;
		}
		~Bank(){}
		
		int getBalance()
		{
			return balance;
		}
		
		bool withdraw(int howMuch)
		{
			if(balance < howMuch)
				return false;
			else
				setBalance(balance - howMuch);
			return true;
		}
};

int main()
{
	Bank b;
	cout << b.getBalance() << endl;
	b.withdraw(400);
	cout << b.getBalance() << endl;
	
	return 0;
}

Static objects

A static variable is a variable that won't be deleted exiting a method (its address is reserved only once). It means that its value setted in a different method calling won't reset. A variable of this type will still exist even if an instance of the class does not. It is shared by all objects.

The same rule applies to methods - we can call them using the class directly without creating its instance. We create static objects by adding the static keyword before their type.


#include <iostream>
using namespace std;

class Example
{
	public:
		Example(){}
		~Example(){}
		
		static void method()
		{
			static_variable++;
		}
		static int static_variable;
};

int Example::static_variable = 0; // "static_variable" definition

int main()
{
	Example x;
	x.method();
	Example::method();
	cout << x.static_variable << " " << Example::static_variable << endl;
	
	return 0;
}
