Decorators

A decorator is a design pattern that allows us to add new functionality to an existing object without modifying its structure. A design pattern is a reusable solution to a commonly occurring problem in software design.

Class methods

Class methods are bound to the class, not to individual objects like regular methods. We create them by writing a @classmethod decorator before them. They also have to take the cls argument (not self). It works the same as self, except that self refers to an object of a class, and cls refers to the class itself. These methods are used when there is a need to provide a constructor method other than __init__() (e.g., when there are many arguments, and we do not want their list in __init__() to be extensive for the sake of code clarity). Class methods don't have access to the fields (objects with self), but they can refer to class attributes (via cls or the class name). Class attributes are, in other words, static variables (they do not contain self and are outside methods).


class Example:
	x = 2
	def method(self):
		self.y = 3
		print(self.y)

	@classmethod
	def classMethod(cls):
		print(Example.x)
		print(cls.x)

e = Example()
e.method()
e.classMethod()
# Example.method() - this won't work
Example.classMethod()

Static methods

We can call static methods using the class directly without creating its instance. They take neither a self nor a cls argument, which makes them essentially ordinary functions within the class. They have no access to the class's objects. They are hardly ever used because it is much easier to create a regular function instead.


class Example:
	def method(self):
		print("Method")

	@staticmethod
	def staticMethod():
		print("Static method")

e = Example()
e.method()
e.staticMethod()
Example.staticMethod()

Abstract classes

An abstract class is a class that is designed specifically to be used as a base class. It can't call out its own objects, so we can only use them in its child classes. An abstract method is a method that is in an abstract class, and has the @abstractmethod decorator in front of it. We create abstract classes by inheriting the ABC object from the imported library. These classes provide a base for subclasses to extend and implement the abstract methods, and are used as a blueprint for other classes. An abstract class forces all its child classes to overwrite all of its abstract methods.

		
from abc import ABC, abstractmethod

class Animal(ABC):
	@abstractmethod
	def voice(self):
		pass
	def voice2(self): # it isn't mandatory to overwrite a non-abstract method from an abstract class
		pass	
					
class Cat(Animal):
	def voice(self):
		print("Meow!")
		
cat = Cat()
cat.voice()

@property, @setter, @deleter

@property is a decorator that encapsulates instance attributes and provides a property. Notice that, in the example below, I didn't write any parentheses while calling this method. It is because a method marked with @property truly becomes a property (something like a variable).

A getter is a method that is used to return values outside of a class, and a setter is a method that edits an object of the class. We should use them because, inside them, we can add, e.g., validation rules. @property is the getter decorator (it returns the "property"), for which we can create a setter decorator (the names of these methods have to be the same). One of these methods will execute accordingly to the performed operation. For example, the setter is called automatically when the "property" is edited.

We can also create deleters, which are destructors, but not for the whole class. We create them in association with the @property method (like the setters), and they "destroy" only the object in interest.


class Example:
	def __init__(self, x):
		self._x = x

	@property
	def x(self): # a getter
		return self._x

	@x.setter
	def x(self, x): # a setter
		self._x = x

	@x.deleter
	def x(self): # a deleter
		del self._x

example = Example(5)
print(example.x)

example.x = 10
print(example.x)

del example.x
