Basics

Fundamental rules

Java counts from zero, not from one.

Quotation marks and apostrophes mean something slightly different, and I will explain it later.

In Java, a semicolon is placed after each command (not everything is a command). We also have to indicate which commands belong to, e.g., a loop, by putting them in braces (unless it's only one line).

We can create comments by using the // symbols. Things that we write after these signs don't count as code. Comments are often used to describe the code. Multiline comments can be written between /* and */ signs.

Hello World program

The System.out.println command is used to display data of all types on the screen. You will learn what the rest of the code means when you get to the lesson about OOP, and for now, let's just say it's a template for all commands.


public class Main
{
	public static void main(String[] args)
	{
		System.out.println("Hello world!");
	}
}

Variables

A variable is a "drawer" for data in the RAM. We can store every sort of data in them. We have a lot of variable types:

byte (1 byte) - integers in this range: [-128 to 127]

short (2 bytes) - integers in this range: [-32768, 32767]

int (4 bytes) - integers in this range: [-2147483648, 2147483647]

long (4 bytes) - integers in this range: [-2147483648, 2147483647]

float (4 bytes) - decimals (up to 6-7 digits after the decimal point)

double (8 bytes) - decimals with double precision (up to 15-16 digits after the decimal point)

String - texts (a number can also be a part of it)

char (2 bytes) - one symbol (one number also counts as one symbol). When using this type, we have to use apostrophes, and not quotation marks because they are assigned to it. This rule goes both ways, and we can't use apostrophes with strings.

boolean (1 byte) - true (all numbers above 0) and false (0) values.

We have to define the type when creating a variable.

Everything that is on the left side of the = sign is a variable's name (it cannot contain spaces), and on the right is a value.

Be cautious because, e.g., variables x and X are not the same!

int number = 4; will initialize an int variable called "number" and immediately assign it the value 4. Java will reserve 4 bytes for it. number = 5; will change the value of this variable to 5.

If we don't immediately have any value to assign to the variable, we can simply declare it, e.g., int number2;. We say that we declare a variable if it is empty at the beginning, and we initialize it when we immediately give it a value. We can assign a value to the declared variable in the same way as if we were changing the value of a variable that already existed (number2 = 6;).


int number = 4; 
number = 5;
int number2;
number2 = 6;

While naming the variables, we cannot add any numbers or spaces at the beginning. The first letter of the variable should not be uppercase. We should write variable names in one of these three ways: prime_number, primeNumber, nPrimeNumber.

We can change a variable to a constant, whose value cannot be changed, by adding final before the variable definition (e.g., final String NAME = "text";). There is an unwritten rule that the names of the constants should be in uppercase.

Variables can be declared in bulk, e.g., int a, b, c;. It is always safer to assign a value to a variable at the beginning to avoid errors (even if its value would be 0). They can also be defined in the same way: int a = 1, b, c = 2; (we can define only some of them if we want). In this case, the b variable cannot be used because it's empty.

If, for example, the number 4 is assigned to a float variabe, it will be written as 4.0. If we divide 4.5 by 2, we get 2.25, and if we divide 5 by 2, we get because when we divide an int by an int, we get an int. When dividing int by int, we should write, e.g. 5.0 / 2, to get the exact result.

If we assign a char character to an int, it will be converted to the ASCII number corresponding to that character. A float number assigned to an int would be abbreviated without rounding (e.g. 2.6 would be equal 2), and an int assigned to a float would be extended (e.g. 2 would be equal 2.0).

We can concatenate strings with the + operator, e.g., String a = b + c;, where b and c are String variables.

When we use long and float types, we have to add a corresponding letter at the end of the number: long a = 1234334124252345252L;, float b = 4.75f;.

If we divide 1234 by 10, the result will be 123 because we are dividing by 10, and not 10.0 (numbers after the decimal point will not be included).

\n symbolizes the enter key, and can be used inside of strings. It will make an enter in the console.

Retrieving data from the user is more complicated, so I will show you how to do it later.

Type casting

Type casting is, in other words, type conversion (e.g., int to double).


double x = 5.5;
int y = (int)x; // the type that we write inside the parenthesis is the one that we are converting to
System.out.println(y);
